# Console TCP Chat

## Требования

- .NET 8
- PostgreSQL (уже запущен на localhost:5434)
- Для миграций: `dotnet-ef` (`dotnet tool install --global dotnet-ef`)

## Подключение к БД

Server подключается к PostgreSQL по строке из `Server/appsettings.json`: хост **localhost**, порт **5434**, БД **ChatApp**, пользователь **admin**. БД должна быть уже запущена — дополнительно поднимать ничего не нужно.

Если БД ещё не запущена, можно поднять её через `docker-compose up -d` (см. docker-compose.yml в корне решения).

## Миграции

При первом запуске или после изменений схемы применить миграции из папки Server:

```bash
cd Server
dotnet ef database update
```

## Запуск

**Server:**

```bash
dotnet run --project Server
```

Сервер слушает порт **5001** (настраивается в `Server/appsettings.json`, секция `Server:Port`).

**Client:**

```bash
dotnet run --project Client
```

По умолчанию клиент подключается к 127.0.0.1:5001. Адрес и порт можно задать аргументами:

```bash
dotnet run --project Client -- --host 127.0.0.1 --port 5001
```

## Проверка сценариев

Ручная проверка по чеклисту. Параметры лимитов и форматов заданы в конфиге Server (`Server/appsettings.json`) и в коде (LoginAttemptTracker).

### 10.2.1 Регистрация, вход, отправка текста (два клиента)

- [ ] Запустить Server и два экземпляра Client (два терминала).
- [ ] В первом клиенте: пункт меню **1** (Register) — ввести логин и пароль; затем **2** (Login) — тот же логин/пароль.
- [ ] Во втором клиенте: **2** (Login) — другой пользователь (предварительно зарегистрировать его из первого клиента или заново Register во втором).
- [ ] В чате указать получателя: `/to <Guid>` (Guid получателя взять из приглашения или из вывода при входе/регистрации).
- [ ] Отправить текстовое сообщение (просто ввести строку и Enter).
- [ ] На втором клиенте должно появиться: `[SenderId]: message`.

### 10.2.2 Отправка и приём файла

- [ ] Разрешённые расширения: **.jpg**, **.png**, **.pdf** (конфиг `FileStorage:AllowedExtensions`). Максимальный размер: **50 МБ** (`FileStorage:MaxFileSizeMb`).
- [ ] На отправителе: `/file <путь>` (путь с пробелами — в кавычках, например `/file "C:\My folder\doc.pdf"`).
- [ ] На получателе: при старте приёма — строка `[file] filename (from SenderId)`; после сохранения — `[file] filename saved to <path>`.

### 10.2.3 Неудачный логин и блокировка

- [ ] После **5 неудачных попыток** входа (неверный пароль) по одному IP или по одному логину сервер возвращает **Blocked** (сообщение о блокировке).
- [ ] Окно учёта попыток: **5 минут**. После истечения окна попытки снова разрешены.
- [ ] Проверка: ввести неверный пароль 5 раз подряд при логине — убедиться в сообщении о блокировке.

### 10.2.4 Лимит соединений

- [ ] В конфиге: **MaxConnectionsPerIp = 3**, **MaxConnections = 100** (секция `Limits`).
- [ ] С одного хоста допускается не более 3 одновременных подключений. Четвёртая попытка подключения должна завершиться отказом (ошибка подключения).

### 10.2.5 Отказ в приёме файла

- [ ] Файл с неразрешённым расширением (например **.txt**, **.exe**) — сервер возвращает Error.
- [ ] Файл размером **больше 50 МБ** — сервер возвращает Error.
- [ ] Проверка: выполнить `/file` для .txt и для файла > 50 МБ — в обоих случаях ожидать сообщение об ошибке.
